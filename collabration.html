Why Automotive Cybersecurity Matters

Cybersecurity is no longer an optional IT function—it is a core engineering, safety, and regulatory requirement across the full vehicle lifecycle.

The Modern Vehicle Context

Modern vehicles are complex distributed systems relying on Connected ECUs, Cloud Platforms, and OTA Updates. These systems are directly linked to critical outcomes:

  • Vehicle Safety & Occupant Protection
  • Data Privacy & User Protection
  • Fleet Reliability & Infrastructure
  • Public Trust in Mobility

Four Pillars of Automotive Security

1. Organizational Governance (CSMS)

Cybersecurity Policies, Risk Management Strategies (TARA), Supplier Audits, and Process Definition.

2. Vehicle On-Board Systems

Network Security (CAN/LIN/Ethernet), Intrusion Detection (IDPS), and Secure Diagnostics.

3. Vehicle Off-Board Systems

Cloud Infrastructure, Telematics Security, Mobile App Protection, and Secure OTA Mechanisms.

4. ECU & Embedded Eng.

Secure Boot, Hardware Trust Anchors (HSM), Cryptographic Review, and Component Pen-Testing.

Regulatory Alignment: India & Global

CIRT’s Automotive Cyber Security Division supports national implementation, technical enablement, and capacity building for evolving regulations.

Global Standards (UNECE)

UNECE R155

Cyber Security Management System (CSMS) requirements for vehicle type approval.

UNECE R156

Software Update Management System (SUMS) governing OTA processes.

Indian Standards (AIS)

AIS-189

Indian standard for Cyber Security Management System implementation.

AIS-190

Indian standard for Software Update & SUMS requirements.

About CIRT
